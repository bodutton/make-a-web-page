Unit 1: Basics of the Web and HTML

How the Web Works

The Web, or internet, is a global network of computers that connects billions of users worldwide. Invented in the 90s, the Web has grown to host over 30 billion pages. The vast majority of the Web is made up of inter-linked hypertext documents written in a language called Hypertext Markup Language, or HTML. When you visit a website (www.example.com) you're actually sending an HTTP request from your computer to a server who in turn finds the appropriate HTML document and sends it back to your computer where your web browser interprets and displays it. This video does a good job of explaining more.


HTML

HTML Hypertext Markup Language makes up most of the content on the web. HTML dictates Text Content, what you see, Markup,what it looks like behind the scenes References to other documents, images and videos Links to other pages in the form of hyperlinks.
Plain text equals plain text, markups do everything else. This video gives a good overview.


Tags and Elements

HTML is written in sentences or elements composed of tags and text. Tags are the markers that tell the text or source how to act or look.
This video explains the distinction well.
There's a tag for anything you want to accomplish just a quick google search away. All tags fall into one of two categories based on how they interact with the page, inline or block.


List of Beginner's Tags

Beginners Tags:
  • Inline
  • v b v = Bold
  • v em v = Italics
  • v a v = Anchor (add hyperlinks, needs attribute like href)
  • v img v = Image (no closing tag, needs source and alt text)
  • v br v = Line Break (no closing tag)
  • v q v = Short Quotation
  • v strong v = Defines Important Text
  • Block
  • v p v = Paragraph (can give heigth and width)
  • v body v = Body
  • v form v = User Input Field

!DOCTYPE HTML

Simple HTML Document Structure:
  • Doctype Tag - v !doctype html v
  • HTML opening tag - v html v
  • head opening tag - v head v
  • title - v title v TITLE! v /title v
  • /head closing tag - v /head v
  • body opening tag - v body v
  • majority of content - v b v content v /b v
  • body closing tag - v /body v
  • /HTML closing tag - v /html v

Unit 2: HTML Structure

HTML

We already know that HTML is the standard markup language used to create web pages. HTML provides the structure and the content for the pages we see on the web. It's its own language with syntax and rules. Each document or page is made up of lines of code called elements marked with tags explaining what it's supposed to do.
Think of a website as a house. Think about the structure, the wood beams holding it all up. Think about and layout of the house, where all the rooms are and how many bathrooms it has. Think about the wiring and plumbing. Thats HTML.


CSS

If HTML is the structure of the house, CSS is the style and decoration. Type of furniture, color of bedrooms, wall hangings, etc. CSS stands for Cascading Style Sheets and it's used for describing the look and formatting of a document written in a markup language. Uses it's own language and syntax to alter font size and color, background position, etc.
CSS gives programmers a way to control the style of related HTML elements. You can dictate commands like "make all class='concept-title' a certain color and size" and manage the look and style of your website.


Developer Tools

The Developer Tools in Google Chrome give us great insight into how a page is structured and provides us not only with detailed information about how a web page is structured, styled, etc. but also empowers us with the ability to alter it. We can add/delete lines of code, adjust pixel ratios, colors, text, images. We can move the existing panes around. It turns any website into a sandbox.

To access Developer Tools: Google Chrome > Any website > Customization Tap (square with 3 bars in the top right-hand corner) > More Tools > Developer Tools
OR
Right Click any link or image > Inspect Element

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.


Trees of Squares

There are two things to remember right off the bat that will make evertything easier.
Rule #1: Every things is a box.Every element of a website is a box and often it's contained within another box. This is accomplished by putting elements inside of eachother in the code. Like you're placing your ships before a game of Battleship, you can place different elements around the page using the grid of pixels to arrange and line them up.
Rule #2: The structure of any website will resemble a big branch growing directly out away from the trunk of the tree. Every website is designed in a branching series of boxes that unfold like nesting dolls from biggest to smallest.
This video explains it perfectly.

Unit 3: Adding Style with CSS

What is CSS?

CSS, or Cascading Style Sheets, is code written to control the style of HTML elements. Think of it as the decorations or the style of your house.

Syntax and Rules

In CSS you will first write a selector that defines what elements of the page a particular style will apply to. Choose one HTML tag or class and alter the behavior of all matching tags on the page. If, for example, you wanted to alter all of the text in div class="units" you would type "div.units {and fill in your commands here}" All phrases are written as follows {attribute:value;} Eveything tagged as a "unit" will now act the same way. To refer to a "class" just add a period and the class name (ex. class=footer becomes .footer{:;})

Avoiding Repitition

Avoiding Repitition is essential in writing good code for a myraid of reasons. Here are three:

Avoiding Errors-An automated system is far less likely to fail. Most mistakes are due to human error, remove it.

Consistency-By working from generic rules down to specifics, the chances that your website will maintain the same style and sizes throughout are much higher. If everything's linkes it's harder for one small part to get out of whack.

Efficiency-Any time you can cut out repitition, meanial tasks, time wasters, etc. you will save time, and we all know that time is money.
